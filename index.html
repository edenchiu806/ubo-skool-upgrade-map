01 / Positioning

Skool 的新定位

這一輪最重要的不是命名漂亮，而是把 `Skool 到底在交付什麼` 說清楚。 新版本只保留真正成立的角色，其他都往後移。

它不是

什麼都有的內容宇宙，也不是一進來就端出所有 AI、顧問、財務與系統的全包承諾。

它真正是

一張會跟著會員升級的一人事業地圖，先幫他起步，再幫他看到下一層。

它要完成

前 48 小時有感、前 7-30 天做出版本並拿到市場訊號、之後逐步解鎖更深資源。

凱龍定版：Skool 不是讓人進來學一堆，而是讓人先做出第一個可賣版本，再沿著一人事業地圖逐步升級。

02 / Principles

底層原則

清晰路徑大於大量內容

每個等級只應該看到此刻最需要的內容與工具，不讓資訊量破壞行動。

先做再懂

先讓會員做出第一個版本，之後再回頭補底層邏輯與系統觀。

價格與進度分離

價格階梯回答買哪一層，進度等級回答他走到哪一層，不能混成一張圖。

Persona 是推薦，不是分裂

Persona 只負責入口分流與選修推薦，不把課程拆成四條平行世界。

前台只講這一層能交付什麼

不要把未來才會解鎖的顧問、財務、系統能力一次性講成前台 promise。

升級感必須真實存在

每一次 level 提升，都要對應到更明確的結果、更深的資源或更大的操作權限。

03 / Three Systems

先分清楚三套系統

這三件事如果不拆開，Skool 很容易又回到「看起來很大，但實際很亂」的狀態。

系統 A｜價格階梯

回答會員現在買哪一層、月費與高價怎麼銜接，以及何時該升價與升級。

  • 創始會員 / 公開月費
  • implementation
  • scale / deployment

系統 B｜會員進度等級

回答這個人現在走到哪、這週該完成什麼、下一層門檻是什麼。

  • 48h 是否啟動
  • 7 天是否做出第一版
  • 30-90 天是否長出訊號與案例

系統 C｜課程與工具解鎖

回答各等級能看到哪些內容、哪些工具現在該開、哪些模組必須後移。

  • 課程主線
  • 模板與工具
  • Persona 選修包

04 / Levels

新的等級架構

凱龍建議保留兩層命名：`前台稱號` 給會員感受升級感，`後台狀態` 給營運團隊判讀進度。

Level 前台稱號 後台狀態 這一層的核心任務
L0 觀望者 旁觀者 被內容吸引，決定要不要進場。
L1 啟動者 啟動者 快速有感，知道自己的起步點。
L2 創造者 建構者 做出第一個可賣版本。
L3 解鎖者 測試者 把版本丟去市場，並開始理解底層邏輯。
L4 實戰者 訊號者 把初步市場反應變成案例與迭代。
L5 專精者 升級候選 選一條更深的能力線，開始系統化。
L6 導航員 放大者 從個人成果走向系統、團隊與放大。

05 / Curriculum & Unlocks

每個等級會拿到什麼

L1 啟動者

先有感，不先上很多課。

Onboarding + Wow

會員會拿到

  • Welcome / Start Here
  • 社群地圖
  • Persona 辨識問卷
  • 個人化 Day 1 體驗一份
  • 48h checklist
  • 一份可分享的社群素材

這一層要解的感受

我不是沒料，只是以前沒有人幫我整理。這裡不是內容庫，而是一個能幫我先走出起點的地方。

L2 創造者

做出第一個可賣版本。

First Sellable Version

會員會拿到

  • 第一個可賣版本 Sprint
  • 版本盤點與切口整理
  • 標題 / 對象 / 價值主張 / CTA 模板
  • 攻略包或一頁式模板
  • 輪播圖生成器
  • AI 擴充內容與首版頁面生成
  • 基礎名單承接機制

前台建議用語

`定位診斷卡` 往後台留，前台改成 `起步盤點`、`可賣亮點卡`、`首發版本方向`、`首版骨架`。

L3 解鎖者

做過之後，開始懂為什麼。

Market Test + Core Theory

會員會拿到

  • 7 天核心課程
  • 三大板塊總論：事業 / 關係 / 健康
  • 事業主線：專業 -> 生意 -> 財富
  • AI 數位分身
  • 個人知識庫引擎
  • CRM 基礎版
  • 市場測試任務與回報模板

這一層的價值

不是一次把所有理論講完，而是讓會員在做過第一版之後，開始理解商業模式、信任、成交與長線結構。

L4 實戰者

把初步市場回應變成穩定訊號。

Signals + Electives

會員會拿到

  • 進階技能樹
  • Persona 專屬選修包
  • LINE Bot 串接
  • 商業分析組
  • Proof Wall 參與權
  • 每月回顧與下一步建議

Persona 選修原則

Boss 走企業 AI 轉型包；Artisan 走手藝人數位加盟包；Builder 走創造者工具箱；Explorer 不做獨立包，回到主線後再轉入其他方向。

L5 專精者

把能力做成自己的系統。

Specialization

會員會拿到

  • 完整天賦樹
  • Skill Chat（全 Agent）
  • Skill 開發權
  • 更深的顧問與工作流整合
  • implementation 適配檢視

這一層的重點

不是再多學幾招，而是把已經跑出來的東西整理成可複製的方法與系統。

L6 導航員

從個人成果走向系統、團隊與放大。

Scale + Deployment

會員會拿到

  • Agent 自訂權
  • 白牌與授權能力
  • 導師面板
  • 社群共建權限
  • 放大與部署視角

這一層的結果

會員開始經營的，不再只是單一作品，而是一個可以部署、協作與複製的資產體系。

06 / Journey

參與者使用者旅程

Skool 不只是放內容的地方，而是一段有節奏的旅程。這裡最值得保留的，是 `48h 啟動 -> 7 天第一版 -> 30 天案例 -> 90 天升級判斷` 這條骨架。

前 48 小時

先被整理清楚，不先看很多內容。

  • Welcome / Start Here
  • Persona 問卷
  • 個人化 Day 1 體驗
  • 48h checklist

前 7 天

先把第一版做出來，並做第一次對外測試。

  • Build Sprint
  • 首版頁面 / 攻略包
  • CTA 與承接點
  • 第一次測試回報

前 30 天

把成果、進度與訊號累積成續費理由。

  • 每週固定 rhythm
  • Proof Wall
  • 進度回報
  • 案例資料提交

前 90 天

進入選修、重估 Persona，並判斷更深升級。

  • Persona 重新評估
  • 進階技能樹推薦
  • implementation 適配
  • 下一階段放大點

旅程強點：這條路徑的價值不在內容多，而在於它把 activation、proof、retention 與 upgrade 串成一條能被營運的節奏。

07 / Persona

Persona 怎麼進來，但不讓課程分裂

凱龍定版的原則很單純：入口分流，主線統一，選修再分流，最後回歸統一。

L1 入口分流

Boss / Artisan / Explorer / Builder 四條起手式，只負責第一次 wow 與起步感。

L2-L3 主線統一

全員回到同一條課程主線：做出版本、理解事業邏輯、開始測市場。

L4-L5 選修分流

依 Persona 與行為數據調整推薦權重，但不變成四套獨立社群。

L3 結束重估

Persona 可以流動。它是推薦引擎，不是身分鎖定。

重要邊界：Explorer 不需要獨立進階課程包。文件裡最穩的結論，是讓他先走主線，之後再轉入 Boss / Artisan / Builder 路徑。

08 / Boundaries

哪些東西不能過早承諾

L1-L3 不該當前台 promise 的內容

  • 全 Agent Skill Chat
  • Skill 開發權
  • Agent 自訂權
  • 白牌授權
  • 導師面板
  • 完整財務系統
  • 全部顧問資源一次開放

正確做法

  • 先交付第一階段結果
  • 再依 level 與標籤逐步解鎖
  • 把升級感建立在真實可用的新資源上
  • 讓前台語言只對應現在看得到的結果

09 / Structure

Skool 內部版面建議

內部版面不需要太多層，重點是讓新會員知道先去哪、做完後去哪、下一層又在哪裡。

Welcome / Start Here

先做 Persona 問卷、拿到 Day 1 體驗、完成 48h checklist。

Build Sprint

放本週唯一主題、模板、範例、公改與交付規格。

Core Map

放 L3 的核心課程與一人事業主線，不把所有選修都堆進來。

Tools + Templates

只顯示已解鎖工具，讓每個新模組出現都像打開新地圖。

Proof Wall

放 before / after、案例、市場訊號，讓續費理由可見。

Next Stage

只顯示當前適合的下一層，不提前把所有高價與高權限混在首頁。

10 / Sources & Advisors

依據與顧問統整

這一版不是無中生有，而是由凱龍回收既有規格，並交給三路專家分別審核 `課程架構`、`使用者旅程`、`模組解鎖` 後整合而成。

課程架構顧問

保留 `清晰路徑 > 大量內容` 與 `先做再懂`，並確立 L1-L3 主線、L4 之後才進技能樹。

旅程顧問

保留 `48h -> 7 天 -> 30 天 -> 90 天` 的骨架，並把 proof 與續費理由放回核心位置。

解鎖顧問

確認 Persona 是入口分流與推薦權重，不是四套課程；高階模組應後移解鎖。
